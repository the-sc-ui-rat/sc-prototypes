GSD — Get Shit Done

What is GSD?

GSD is a spec-driven, context-engineered development system built for Claude Code. It solves "context rot" — the quality degradation that happens as Claude fills its context window — by giving every agent session a fresh 200k context loaded from structured docs.

BMAD vs GSD

BMAD plans the project (Analyst → PM → Architect → PO → SM). GSD executes it. They complement each other: use BMAD to create the docs, then hand them to GSD to build story by story in parallel waves.

The core concept — context engineering

GSD keeps 4 structured documents always loaded into Claude's context. Every new session reads these first. Claude always knows: 1) What is this project? 2) What has been done? 3) What needs to be done? 4) What are we doing right now?

PROJECT.md

Mission, stack, design tokens, constraints, key people. The permanent project bible. Rarely changes.

REQUIREMENTS.md

Scenarios, flows, acceptance criteria, non-functional requirements. Source of truth for what must be built.

ROADMAP.md

Milestones → Epics → Stories. The sequence of work. Stories are atomic tasks deliverable in hours, not weeks.

STATE.md

Current milestone, current epic, what's done, what's next, decisions log, blockers. Updated after every story.

The 4-step execution cycle

Every story follows the same cycle. Claude runs it autonomously — you only intervene at Pause Gates (irreversible actions) or when blocked.

Step 1

Discuss

Read the story file. Cross-check against PROJECT, REQUIREMENTS, ROADMAP. Summarise: goal, acceptance criteria, file scope, implementation plan. Do NOT write any code until intake is complete.

Step 2

Plan

Research approaches. Create an atomic task plan in XML format. Group tasks into dependency waves. Verify the plan passes quality checks before execution begins.

Step 3

Execute (Waves)

Run plans in parallel waves based on dependencies. Each wave gets a fresh 200k token context. Independent tasks run simultaneously. Sequential tasks wait for prerequisites. Atomic git commit after each story.

Step 4

Verify

Run lint, type-check, build. Self-review against acceptance criteria. Invoke qa-validator agent. Fix all blocking issues. Only qa-validator can mark a story Done — Claude cannot self-certify.

Workflow #1 — Setup GSD documents

Step 1: Create project folder in GitHub

  • Create folder in the relevant repo (e.g. design-team-claude/projects/)
  • GitHub requires at least one file — start with README.md
  • Commit and push to main

Step 2: Create the 4 GSD docs

  • docs/PROJECT.md — mission, stack, tokens, constraints
  • docs/REQUIREMENTS.md — scenarios, flows, acceptance criteria
  • docs/ROADMAP.md — milestones, epics, stories
  • docs/STATE.md — current progress, decisions, blockers

Step 3: Load docs into every session

  • Start every Claude Code session from the project folder
  • Say: "Read PROJECT.md, REQUIREMENTS.md, ROADMAP.md, and STATE.md first"
  • Claude now has full context — no re-explaining needed

Step 4: Update STATE.md after each story

  • Tick off completed stories in ROADMAP.md
  • Update STATE.md with: current milestone/epic, next action, decisions, blockers
  • This is what keeps the next session coherent

Workflow #2 — Setup dev environment

Step 1: Initialise Expo project

  • cd projects/quick-secure-auth/native
  • npx create-expo-app@latest . --template blank-typescript
  • Install Expo Go on your iPhone and Android test device

Step 2: Configure SC design tokens

  • Create src/tokens.ts with SC colour values
  • Use StyleSheet.create() — no Tailwind on native
  • Web version can use Tailwind CSS 4 with tokens.css

Step 3: Launch Claude in project folder

  • cd ~/Desktop/design-team-claude/projects/quick-secure-auth
  • Type: claude
  • Say: "Read all 4 GSD docs and confirm you have context"

Step 4: Run on device via Expo Go

  • npx expo start → QR code appears in terminal
  • Teammates open Expo Go app → scan QR → live prototype on device
  • Hot reload — changes appear instantly, no rebuild

Workflow #3 — Build stories in waves

Step 1: Pick the next story from ROADMAP.md

  • Stories are atomic — completable in hours, not days
  • Each story has: goal, acceptance criteria, file scope, tasks
  • Sequence is strict — complete each before starting the next

Step 2: GSD execution loop prompt

  • "Read PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md. Then implement Story [X]."
  • Claude runs Discuss → Plan → Execute → Verify autonomously
  • Only stops at Pause Gates or blockers

Step 3: Wave execution

  • Independent tasks run in parallel (multiple agent calls)
  • Sequential tasks wait for prerequisites
  • Each wave gets a fresh context window — no rot

Step 4: Atomic git commit + STATE update

  • Commit immediately after each story is Done
  • Message format: "feat: [story title] (Story X.X)"
  • Update STATE.md — tick story, update current position, log decisions

Workflow #4 — Share & deploy

Web → Vercel (desktop sharing)

  • Login to vercel.com with GitHub account
  • Connect design-team-claude repo → select projects/quick-secure-auth/web as root
  • Push to main → auto-deploys. Share URL with stakeholders

Native → Expo Go (device sharing)

  • npx expo start from native/ folder
  • Share QR code or tunnel URL with teammates
  • They install Expo Go → scan → live on their device. No App Store.

Biometric simulation (Face ID)

  • expo-local-authentication triggers real Face ID / fingerprint
  • Feels authentic in demo — stand-in for Oloid until engineering integration
  • NFC simulation = tap gesture with haptic feedback

Context7 MCP — stay current

  • Claude uses Context7 to pull latest Expo / React Native docs
  • Prevents using deprecated APIs (Expo updates frequently)
  • Already configured in your global Claude setup
